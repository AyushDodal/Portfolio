Customer Churn Prediction

Classification Python XGBoost

A full statistical modeling project predicting telecom customer churn. It includes Exploratory Data Analysis and model building (Logistic Regression, Random Forest, XGBoost)

The goal: identify high-risk customers for targeted retention campaigns.

Key Results

  • Baseline Logistic Regression AUC: 0.82
  • XGBoost AUC after tuning: 0.90
  • Top predictors: Contract (month-to-month), tenure, Payment method, Monthly charges

Business Impact

This model enables targeted retention programs by surfacing likely-to-churn customers, guiding interventions such as promotional offers, contract upgrades, or billing alternatives to reduce churn and increase lifetime value.

Technical approach & modeling notes

Data prep: handled blank TotalCharges, encoded binary and categorical features with one-hot encoding, scaled continuous features (MinMax).

Modeling: baseline logistic regression for interpretability, Random Forest and XGBoost for performance. Used stratified 80/20 split and cross-validation for tuning.

Metrics: accuracy, precision, recall, F1, ROC-AUC. Churn class is ~26% — used class weighting and calibration where needed.

Data: Telco Customer Churn (Kaggle)

Tools: Python, scikit-learn, XGBoost, Power BI
